## v0.10.30 Change

v0.10.30 hardens the daily publish chain.

The problem:

```text
The daily Action can collect useful flow and rotation data, but a partial web
publish is dangerous: the site may show sector rotation while missing the
module review, ETF readiness gate, or data reality audit.
```

The solution:

```text
Add a published-data completeness guard before Worker build.
If any required web JSON is missing or violates its basic contract, CI fails.
```

## New Guard

Command:

```bash
npm run validate:data
```

scripts/validate-published-data.mjs
